title: CRD reference
description: Field-level reference for the Agent, AgentFleet, and ModelPool custom resources in agents.x-k8s.io/v1alpha1.
---

The three custom resources in **`agents.x-k8s.io/v1alpha1`**, field by field. The
source of truth is `charts/agentctl/crds/` (generated by `agentctl-crdgen`); for
the conceptual tour see [the CRDs](/docs/concepts/crds).

## Agent

`Agent.spec` — one logical agent rendered to a workload whose shape follows `mode`.

| Field | Type | Notes |
|---|---|---|
| `mode` | enum `once`/`loop`/`reactive`/`schedule` | **required**; determines the workload kind |
| `image` | string | the conformant-agent image; **required iff `classRef` unset**, forbidden when set |
| `classRef.name` | string | reference to an `AgentClass` (image + ops profile); mutually exclusive with `image` |
| `instruction` | string | required for non-reactive modes |
| `subscribe` | string[] | reactive subscriptions (MCP resource URIs); required for `reactive` |
| `model` | string | declared model id (metadata; printer column) |
| `modelPool` | string | declared capability: the [`ModelPool`](#modelpool) to bind; admission-validated |
| `intelligenceRef.name` | string | reference to an `IntelligenceService`/ModelPool |
| `mcpServerSetRefs[].name` | string[] | reusable MCP server bundles |
| `exec` | bool | declared capability; trifecta leg |
| `egress` | bool | declared capability; trifecta leg |
| `secrets` | string[] | namespace-local Secret names the agent may read; trifecta leg |
| `substrate` | enum `stock-unix`/`kata-hybrid`/`sidecar-emptydir` | [substrate tier](/docs/concepts/substrate); not hard-defaulted |
| `surfaces.{management,metrics,a2a}` | bool (default false) | which control-plane surfaces to expose |
| `loop.interval` / `loop.deadline` | string | loop cadence (e.g. `5m`, `24h`); `loop` only |
| `schedule.cron` / `schedule.timezone` | string | CEL-required when `mode: schedule` |
| `limits.{maxTokens,maxSteps,maxDepth,treeTokenBudget}` | uint64 | per-agent budgets |
| `access.public` | bool | doc-only in v1 (records intent) |
| `access.oidc` | object | per-agent [OIDC](/docs/guides/security/oidc) — see below |

**`access.oidc`** — `issuer` (required), `audiences[]`, `jwksUri` (override
discovery), `forwardIdentity` (bool), `requiredClaims[]` (each `{claim, anyOf[]}`,
**all** must hold).

**CEL validation:** `schedule` is required when `mode: schedule`.

`Agent.status` — a curated projection, never a raw manifest dump:

| Field | Notes |
|---|---|
| `conditions[]` | taxonomy: `Validated`, `Rendered`, `Ready`, `Draining`, `Degraded`, advisory `TrifectaUnionObserved` (mirrors `metav1.Condition`) |
| `contract.{agentVersion,contractVersion,mode}` | read from the live manifest |
| `contract.operatorTools[]` | operator tools the agent advertises (read, never assumed) |
| `contract.served.{management,metrics,a2a,events}` | served-surface summary |
| `observedGeneration` | the `.metadata.generation` this status reflects (hot-loop guard) |
| `phase` | coarse human-facing phase |

Printer columns: `Mode`, `Model`, `Ready`, `Phase`, `Age`.

## AgentFleet

`AgentFleet.spec` — a replicated, autoscaled set. **KEDA owns `.spec.replicas`** in
claim mode, so the rendered workload omits it.

| Field | Type | Notes |
|---|---|---|
| `template` | `Agent`-spec-shaped | the per-replica agent definition |
| `replicas` | uint32 | claim-mode target of the `scale` subresource; KEDA-owned in steady state |
| `scaling.mode` | enum `claim`/`shard` | elastic (KEDA) vs fixed partition |
| `scaling.min` / `scaling.max` | uint32 | claim-mode KEDA-elastic range; unset min ⇒ scale to 0 |
| `scaling.shards` | uint32 | shard-mode partition count `N` (FNV-1a/64 modulus + StatefulSet replicas) |
| `scaling.target.signal` | string | **contract-neutral** metric token (mapped onto the metrics schema) |
| `scaling.target.value` | string | per-replica target value KEDA scales toward |
| `workSource` | string | the work source URI |

Renders to a `Deployment` (claim) or `StatefulSet` (shard). See
[scaling](/docs/guides/scaling).

## ModelPool

`ModelPool.spec` — secretless, budgeted model access for the
[ModelGateway](/docs/guides/intelligence).

| Field | Type | Notes |
|---|---|---|
| `provider` | string | **required**; provider id (`mock`/`anthropic`/`openai`/...) |
| `endpoint` | string | **required**; provider base URL |
| `credentialSecretRef.{name,key}` | string | **required**; Secret holding the provider API key (gateway-injected; agent never sees it) |
| `models[]` | string[] | allowed model ids |
| `defaultModel` | string | used when a request does not pin one |
| `budget.maxTokens` | int64 | total pool token budget; gateway rejects with 429 once exhausted |

**CEL validation:** `budget.maxTokens > 0`; `credentialSecretRef.name` non-empty.

`ModelPool.status.usedTokens` (int64) is the running meter. Printer columns:
`Provider`, `Endpoint`, `Budget`, `Used`, `Age`.
