title: Env convention
description: The Agent Control Contract downward-API environment-variable convention — identity, drain/grace, sharding, intelligence credentials, and lifecycle.
---

The Agent Control Contract defines a downward-API **environment-variable
convention** that the operator wires into every agent pod. The agent reads these
from env **only** (`valueFrom.fieldRef`) and **never calls the Kubernetes API**.
The source of truth is `contract/schemas/env-convention.json` (authority: agent
RFCs 0014/0015/0011/0019).

## Rules

- **All vars are OPTIONAL** — absence is never a config error.
- **Identity is descriptive**, never load-bearing (no placement/scheduling
  decision is made from it).
- **Empty is unset** — an empty value is coerced to unset, not `Some("")`.
- **Secrets travel only the `*_TOKEN[_FILE]` path** — the config file never carries
  a credential.

## Identity (`fieldRef`)

| Var | Source |
|---|---|
| `AGENT_RUN_ID` | minted ULID if unset; stable across a retried Job |
| `AGENT_POD_NAME` | `metadata.name` |
| `AGENT_POD_UID` | `metadata.uid` |
| `AGENT_POD_NAMESPACE` | `metadata.namespace` |
| `AGENT_NODE_NAME` | `spec.nodeName` |

## Drain / grace

| Var | Meaning |
|---|---|
| `AGENT_POD_GRACE_SECONDS` | mirrors `terminationGracePeriodSeconds` (operator-injected) |
| `AGENT_DRAIN_TIMEOUT` | drain budget (also a `--drain-timeout` flag) |

## Sharding

| Var | Meaning |
|---|---|
| `AGENT_SHARD` | `"K/N"` from the StatefulSet ordinal (`K=ordinal`, `N=replicas`) — agentctl owns `K`; the agent does not discover its own shard |
| `AGENT_SHARD_TIMER` | `shard0` (default) or `keyed` |
| `AGENT_STANDBY` | standby-surface toggle (bool) |

## Intelligence credentials

| Var | Meaning |
|---|---|
| `AGENT_INTELLIGENCE_TOKEN` | endpoint #1 credential (secret) |
| `AGENT_INTELLIGENCE_TOKEN_FILE` | endpoint #1 credential file path |
| `AGENT_INTELLIGENCE_TOKEN_{N}` / `_{N}_FILE` | per-endpoint, 1-indexed by endpoint list position |

## Lifecycle (restart-only; never a config-file key)

| Var | Meaning |
|---|---|
| `AGENT_SERVE_MCP` | management transport address (or absent) |
| `AGENT_INTELLIGENCE` | intelligence primary endpoint/transport (e.g. `unix:/run/agentctl/infer/infer.sock` for [routed infer](/docs/guides/security/attested)) |
| `AGENT_MODE` | `once`/`loop`/`reactive`/`schedule` |
| `AGENT_MODEL` | operator-declared model id (never a secret) |

See [provisioning](/docs/guides/provisioning) for how the operator injects these.
