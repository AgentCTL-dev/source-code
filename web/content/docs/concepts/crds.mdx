---

agentctl ships three namespaced custom resources, all in the group
**`agents.x-k8s.io/v1alpha1`**. This page is the conceptual tour; the
field-by-field schema is the [CRD reference](/docs/reference/crd-reference), and
the source of truth is `charts/agentctl/crds/`.

| Kind | Short names | Renders to | Purpose |
|---|---|---|---|
| `Agent` | `agent` | `Job` / `CronJob` / `Deployment` | one logical agent |
| `AgentFleet` | — | `Deployment` / `StatefulSet` | a replicated, autoscaled set |
| `ModelPool` | `mp` | (config for the ModelGateway) | secretless, budgeted model access |

## Agent

One logical agent: an instruction plus bindings, rendered to a workload whose
shape follows `mode`.

```yaml
apiVersion: agents.x-k8s.io/v1alpha1
kind: Agent
metadata:
  name: demo
spec:
  mode: once          # once -> Job | schedule -> CronJob | loop|reactive -> Deployment
  image: ghcr.io/agentd-dev/agentd:1.0.0   # required iff classRef unset
  instruction: "..."  # required for non-reactive modes
  subscribe: ["queue://jobs"]              # required for reactive
  modelPool: mockpool                      # declared capability -> binds a ModelPool
```

Key fields:

- **`mode`** (`once` | `loop` | `reactive` | `schedule`) determines the rendered
  workload kind.
- **`image` vs `classRef`** are mutually exclusive — a classless agent names its
  own image; a class-backed agent inherits the image + ops profile.
- **Declared capabilities** `exec`, `egress`, and `secrets` are the *lethal
  trifecta*: requesting all three is denied by admission unless explicitly
  allowed. `modelPool` is validated for existence/permission. See
  [the admission gate](/docs/guides/security/overview).
- **`access.oidc`** configures per-agent [OIDC](/docs/guides/security/oidc)
  authn/authz for inbound A2A calls.
- **`substrate`** pins the [substrate tier](/docs/concepts/substrate)
  (`stock-unix` | `kata-hybrid` | `sidecar-emptydir`).
- **`status`** is a curated projection of the live capabilities manifest plus the
  [conditions taxonomy](/docs/reference/crd-reference) — never a raw manifest dump.

## AgentFleet

A replicated, autoscaled set of agents. **KEDA owns `.spec.replicas`** in claim
mode, so the rendered workload omits it.

```yaml
apiVersion: agents.x-k8s.io/v1alpha1
kind: AgentFleet
metadata:
  name: workers
spec:
  template:                 # the per-replica Agent definition
    mode: reactive
    image: ghcr.io/agentd-dev/agentd:1.0.0
    subscribe: ["queue://jobs"]
  scaling:
    mode: claim             # claim (elastic, KEDA) | shard (fixed partition)
    min: 0
    max: 10
    target: { signal: pending_events, value: "5" }
  workSource: "queue://jobs"
```

- **`scaling.mode: claim`** -> `Deployment`, elastic 0..max on the
  `scaling.target` signal. **`shard`** -> `StatefulSet` with a fixed `shards`
  count; each replica gets a `K/N` shard identity. See [scaling](/docs/guides/scaling).
- **`scaling.target.signal`** is a contract-neutral metric token the operator
  maps onto the negotiated [metrics schema](/docs/reference/metrics).

## ModelPool

A pool of model access. Agents are **networkless and hold no provider secrets**;
the [ModelGateway](/docs/guides/intelligence) supplies model access through a
credential-injecting, metering, budget-enforcing proxy configured by this CRD.

```yaml
apiVersion: agents.x-k8s.io/v1alpha1
kind: ModelPool
metadata:
  name: mockpool
spec:
  provider: mock
  endpoint: http://mock-provider.default:8080
  credentialSecretRef: { name: provider-credentials, key: api-key }
  models: ["mock-model-v1"]
  defaultModel: mock-model-v1
  budget: { maxTokens: 150 }   # gateway rejects with 429 once exhausted
```

The gateway reads `credentialSecretRef` and injects the key on each request — the
agent never sees it. `status.usedTokens` meters consumption against
`budget.maxTokens`. See [intelligence + budgets](/docs/guides/intelligence).
