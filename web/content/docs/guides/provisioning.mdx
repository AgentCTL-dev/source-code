---
title: Provisioning
description: How the operator and admission webhook turn an Agent or AgentFleet CR into a confined, correctly-wired Kubernetes workload.
---

Provisioning is the path from `kubectl apply` to a running, confined agent. Two
components own it: the **admission webhook** (mutate + validate) and the
**operator** (reconcile to a workload). The worked example is the
[agentd](/docs/getting-started/first-agent) `Agent`.

## The reconcile path

```mermaid
sequenceDiagram
  actor U as kubectl
  participant API as kube-apiserver
  participant ADM as admission webhook
  participant OP as operator
  participant AG as agentd pod
  U->>API: apply Agent (image, mode, modelPool, caps)
  API->>ADM: mutate (defaults) then validate (trifecta + registry)
  ADM-->>API: patched + admitted
  OP->>API: watch Agents
  OP->>API: apply Deployment/Job/StatefulSet (confined + downward env + mgmt socket)
  API-->>AG: scheduled + started
  AG->>AG: serve mgmt MCP on unix socket; idle / reactive
```

## Mode -> workload

The operator renders the workload kind from `spec.mode`:

| `mode` | Workload | Notes |
|---|---|---|
| `once` | `Job` | run to completion; exits with a [contract exit code](/docs/reference/exit-codes) |
| `schedule` | `CronJob` | requires `spec.schedule.cron` (CEL-enforced) |
| `loop` | `Deployment` | re-entry on `spec.loop.interval` |
| `reactive` | `Deployment` | subscribes to `spec.subscribe` work sources |

An [`AgentFleet`](/docs/concepts/crds) renders to a `Deployment` (claim) or
`StatefulSet` (shard) — see [scaling](/docs/guides/scaling).

## The admission gate

Before anything is rendered, the webhook runs:

- **Mutating** — defaults `app.kubernetes.io/*` labels, `mode`, and a minimal
  `surfaces{}` set. It does **not** hard-default `substrate`.
- **Validating** — denies the **lethal trifecta** (`exec && egress && secrets`)
  unless `agentctl.dev/allow-trifecta: "true"`; enforces an image-registry
  allow-list (`admission.allowedRegistries`); checks cross-object `modelPool`
  existence.

See [the security overview](/docs/guides/security/overview) for the full gate.

## What the operator wires into the pod

- **Confinement** — drop-all-caps, `seccomp: RuntimeDefault`,
  `readOnlyRootFilesystem`, no privilege escalation, `runAsNonRoot` where the
  socket allows.
- **Downward-API identity** — `AGENT_POD_NAME`, `AGENT_POD_UID`,
  `AGENT_POD_NAMESPACE`, `AGENT_NODE_NAME` via `valueFrom.fieldRef`. The agent
  reads identity from env only and never calls the Kubernetes API. See the
  [env convention](/docs/reference/env-convention).
- **Management socket** — a per-pod subdir on the [substrate](/docs/concepts/substrate)
  socket root so the node-agent can discover and drive the agent.
- **Capability env** — e.g. `AGENT_INTELLIGENCE` when a `modelPool` is bound (see
  [intelligence](/docs/guides/intelligence)), `AGENTCTL_API_TOKEN` when the
  [API token gate](/docs/guides/security/api-token) is on.

## Status

The operator patches `Agent.status` with the
[conditions taxonomy](/docs/reference/crd-reference) (`Validated`, `Rendered`,
`Ready`, `Draining`, `Degraded`, advisory `TrifectaUnionObserved`) and a curated
`contract` projection read from the agent's live `agent://capabilities` manifest
— the operator drives only what the agent actually advertises (graceful
degradation), intersected with the desired `spec.surfaces`.

## Operator HA

The operator runs active/standby with a `coordination.k8s.io` `Lease`. Readiness
is **not** gated on leadership (every replica serves `/healthz` `/readyz`
`/metrics`), so a `RollingUpdate` never deadlocks; failover is roughly the lease
duration. See [architecture §10](/docs/architecture).
