title: Install on kind
description: Bring up a one-node kind cluster with cert-manager and install the agentctl control plane with Helm.
---

This page gets the agentctl control plane running on a local
[kind](https://kind.sigs.k8s.io/) cluster. The same chart installs unchanged on
any conformant Kubernetes cluster — see the
[Helm chart reference](/docs/reference/helm-values) and the
[operations runbook](/docs/operations) for production notes.

## Prerequisites

| Requirement | Why |
|---|---|
| **kind** + **kubectl** + **Helm 3** | local cluster + install tooling |
| **cert-manager** (>= 1.13) | issues every serving/mTLS cert and injects the `caBundle`s |
| Aggregation layer enabled | the APIServer registers an `APIService` (standard on kind) |
| **Postgres** | the A2A gateway + ModelGateway durable store — bundled by default |

A NetworkPolicy-enforcing CNI (Calico/Cilium) is needed *only* if you later set
`networkPolicies.enabled=true`; kindnet ignores NetworkPolicies. See
[NetworkPolicies](/docs/guides/security/netpol).

## 1. Create the cluster

```console
kind create cluster --name agentctl
```

## 2. Install cert-manager

cert-manager mints the agentctl CA and every serving/mTLS leaf, and rotates them
automatically. Install it and wait for it to be ready:

```console
kubectl apply -f https://github.com/cert-manager/cert-manager/releases/latest/download/cert-manager.yaml
kubectl -n cert-manager rollout status deploy/cert-manager-webhook
```

## 3. Pre-create the namespace

The control-plane namespace needs a relaxed PodSecurity level (the node-agent
uses `hostPath` + `hostPID`), and Helm cannot reliably own the namespace it
installs into, so create it first:

```console
kubectl create namespace agentctl-system
kubectl label namespace agentctl-system \
  pod-security.kubernetes.io/enforce=privileged \
  pod-security.kubernetes.io/warn=privileged
```

## 4. Install the chart

From a checkout of the repository (kind-loaded local images):

```console
helm install agentctl ./charts/agentctl -n agentctl-system
```

Or pull the published chart and images from GHCR (no local image loading):

```console
helm install agentctl oci://ghcr.io/agentctl-dev/charts/agentctl \
  -n agentctl-system --version 0.1.0 \
  --set image.registry=ghcr.io/agentctl-dev --set image.tag=0.1.0
```

## 5. Verify

```console
kubectl -n agentctl-system get pods                          # components Running
kubectl -n agentctl-system get certificate                   # all READY=True
kubectl get apiservice v1alpha1.management.agents.x-k8s.io    # AVAILABLE=True
```

After install, `helm test agentctl -n agentctl-system` runs a connectivity probe
that checks the gateway `/healthz` and that the aggregated APIService is
registered and available.

## What cert-manager wired

A self-signed bootstrap `Issuer` mints the **agentctl CA**; a CA `Issuer` then
mints every leaf — the APIServer serving cert (CA injected into the
`APIService.spec.caBundle`), the admission webhook serving cert, the node-agent
mTLS server cert, and the control-plane mTLS client cert. Renewal is automatic
(`renewBefore: 720h`). See [the certificate fabric](/docs/guides/security/overview).
