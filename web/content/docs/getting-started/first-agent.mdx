With the [control plane installed](/docs/getting-started/install), declare a
single agent. An [`Agent`](/docs/concepts/crds) is one logical agent: an
instruction plus bindings that the operator renders to a workload whose shape
follows `mode` (`once` -> `Job`, `schedule` -> `CronJob`,
`loop`/`reactive` -> `Deployment`).

## A once-mode agent

`mode: once` runs a single task to completion and exits with a
[contract exit code](/docs/reference/exit-codes). This manifest uses the real
CRD group `agents.x-k8s.io/v1alpha1`:

```yaml
apiVersion: agents.x-k8s.io/v1alpha1
kind: Agent
metadata:
  name: demo
  namespace: default
spec:
  mode: once
  image: ghcr.io/agentd-dev/agentd:1.0.0
  instruction: "Read /data/report.md and write a 3-bullet summary to /data/summary.md"
```

Apply it and watch the status:

```console
kubectl apply -f agent-once.yaml
kubectl get agent demo -o wide
kubectl describe agent demo
```

The operator validates the spec (admission runs the
[lethal-trifecta gate](/docs/guides/security/overview) + an image-registry
allow-list), renders a `Job`, and patches `Agent.status` — the
[conditions taxonomy](/docs/reference/crd-reference) (`Validated`, `Rendered`,
`Ready`, ...) plus a curated projection of the agent's live capabilities
manifest. A clean run exits `0` ([`EXIT_OK`](/docs/reference/exit-codes)).

## A reactive agent

`mode: reactive` runs as a long-lived `Deployment` that subscribes to work
sources and serves its management profile over the substrate socket, so the
node-agent can discover it and read its `agent://capabilities`:

```yaml
apiVersion: agents.x-k8s.io/v1alpha1
kind: Agent
metadata:
  name: worker
  namespace: default
spec:
  mode: reactive
  image: ghcr.io/agentd-dev/agentd:1.0.0
  instruction: "serve the management profile"
  subscribe: ["queue://jobs"]
```

## What just happened

```mermaid
sequenceDiagram
  actor U as kubectl
  participant API as kube-apiserver
  participant ADM as admission webhook
  participant OP as operator
  participant AG as agentd pod
  U->>API: apply Agent (image, mode, instruction)
  API->>ADM: mutate (defaults) then validate (trifecta + registry)
  ADM-->>API: patched + admitted
  OP->>API: watch Agents
  OP->>API: apply Job / Deployment (confined + downward-API env + mgmt socket)
  API-->>AG: scheduled + started
  AG->>AG: serve mgmt MCP on the unix socket; run / idle
```

The pod is **confined** (drop-all-caps, seccomp, read-only rootfs) and receives
its identity via the [downward-API env convention](/docs/reference/env-convention)
— the agent never calls the Kubernetes API.

## Drive it: management verbs

Once an agent serves its management profile, the aggregated APIServer exposes
SAR-gated verbs (`drain`, `lame-duck`, `cancel`):

```console
kubectl create --raw /apis/management.agents.x-k8s.io/v1alpha1/namespaces/default/agents/worker/drain
```

The call is proxied through the front-proxy (mTLS), authorized by a
`SubjectAccessReview`, and forwarded to the node-agent, which kernel-attests the
pod (`SO_PEERCRED`) before reaching the agent over the unix socket. See the
[provisioning guide](/docs/guides/provisioning) and
[management path](/docs/architecture).
